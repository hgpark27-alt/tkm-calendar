## 1. Apps Script 프로젝트 생성
1. [script.google.com](https://script.google.com) 접속 (해당 계정으로 로그인 확인)
2. "새 프로젝트" 클릭
3. 프로젝트 이름을 "TKM 캘린더 백엔드"로 변경 (좌측 상단, 선택사항)

## 2. 코드 붙여넣기
1. 기본으로 열려있는 `Code.gs` 파일의 내용을 전부 지우고
2. 이 폴더의 `backend\Code.gs` 파일 내용을 그대로 복사해서 붙여넣기
3. 저장 (Ctrl+S)

## 3. Calendar API 고급 서비스 활성화
1. 좌측 메뉴에서 "서비스" 옆 **+** 클릭
2. 목록에서 **Calendar API** 선택 → "추가"
3. (별도 설정 없이 추가만 하면 됨)

## 4. 테스트 실행
1. 상단 함수 선택 드롭다운에서 `_selfTest` 선택
2. "실행" 버튼 클릭
3. 처음 실행 시 권한 승인 창이 뜸 → 본인 계정 선택 → "고급" → "TKM 캘린더 백엔드(안전하지 않음)으로 이동" → 허용
   - "안전하지 않음" 경고는 구글이 아직 검증 안 한 개인 스크립트라서 뜨는 정상적인 문구입니다.
4. 실행 로그(하단 "실행 로그" 또는 Ctrl+Enter)에 `[]` 또는 이벤트 배열이 찍히면 성공
5. 에러가 나면 그 에러 메시지를 저에게 그대로 붙여넣어주세요 (예: 캘린더 접근 권한 문제인지 API 활성화 문제인지 바로 알 수 있음)

## 5. 웹앱으로 배포
1. 우측 상단 "배포" → "새 배포"
2. 유형 선택(톱니바퀴) → "웹앱"
3. 설정:
   - **실행 계정**: 나 (본인)
   - **액세스 권한**: **전체 공개** (로그인 없이 12명이 호출해야 하므로 반드시 이 옵션)
4. "배포" 클릭 → 권한 재확인 뜨면 다시 허용
5. 완료 후 나오는 **웹앱 URL**을 복사해서 저에게 전달해주세요
   - 형태: `https://script.google.com/macros/s/AKfycb.../exec`

## 6. 브라우저로 직접 확인 (선택, 권장)
웹앱 URL 뒤에 `?action=ping`을 붙여서 브라우저 주소창에 입력:
```
https://script.google.com/macros/s/AKfycb.../exec?action=ping
```
`{"ok":true,"pong":true,"calendarId":"..."}` 가 뜨면 정상 배포된 것입니다.
